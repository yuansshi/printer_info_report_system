# Printer Operations Dashboard

这是打印机邮件报表项目的本地运营界面。当前历史从 2026-06-01 开始，页面使用后台任务发布的静态快照，不直接访问邮箱，也不会修改邮箱状态。

## 数据刷新

```bash
python3 update_printer_dashboard.py
```

该命令补齐缺失日期、重抓最近 3 个完整日、校验历史并原子替换 `dashboard/data.js`。仅从现有历史重建：

```bash
python3 update_printer_dashboard.py --rebuild-only
```

## 本地运行

```bash
python3 -m http.server 4173 --bind 127.0.0.1 --directory dashboard
```

打开 `http://127.0.0.1:4173/`。

## 页面

- `总览`：核心指标、日/周/月通知趋势、客户分布、近期告警和数据质量摘要。
- `设备`：设备搜索、排序、状态筛选和设备详情抽屉。
- `告警`：耗材、服务部件、故障和映射问题明细。
- `计费器`：保留原始 `[1]-[5]` 累计值及设备快照。
- `数据质量`：未知编号、位置缺失、重复映射和日期覆盖。

全局日期、客户、状态和文本筛选作用于总览、设备、告警、计费器以及质量页的问题设备表；质量卡片固定显示整份快照的发布基线。导出按钮下载当前筛选范围的设备 CSV。

## 当前限制

- 页面本身是静态快照；远程调度器每天运行更新命令并发布到前一个完整日。
- 计费器字段尚未按型号映射为黑白、彩色、复印或扫描。
- 当前没有用户认证或服务端 API；历史以可审计的每日 JSON 分区保存，而不是数据库。
